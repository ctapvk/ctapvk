In the secondth exemple, the flex breaks everything: the text is no longer wrapping.

This is NOT a bug: the CSS specs are made like this. Since the box is flexible, it has to take the “min-width: auto”, which is the width if its narrowest child, which is (in this case) the loooooong word.

So, the text, again, overflows.

3)

The fix for the problemating the rose from the 2) is quite simple: since it’s a matter of “min-width”, one has to specify it back to the minimal, thus 0px.

This “min-width: 0px” has to be put on the box that is flexible, aka the box whose parent has “display-flex”.
